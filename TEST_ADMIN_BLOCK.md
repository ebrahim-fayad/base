# دليل اختبار ميزة حظر/حذف الأدمن وتسجيل الخروج التلقائي

## المتطلبات
1. حسابين أدمن على الأقل
2. متصفحين مختلفين أو نافذتين incognito
3. Firebase مفعل ومضبوط بشكل صحيح

## خطوات الاختبار

### الاختبار 1: حظر أدمن متصل (Real-time)

#### الخطوات:
1. **في المتصفح الأول:**
   - سجل دخول بحساب أدمن (مثلاً: admin1@example.com)
   - انتقل إلى أي صفحة في لوحة التحكم
   - اترك الصفحة مفتوحة

2. **في المتصفح الثاني:**
   - سجل دخول بحساب أدمن آخر له صلاحيات (مثلاً: admin2@example.com)
   - انتقل إلى صفحة إدارة المستخدمين/الأدمن
   - ابحث عن admin1
   - اضغط على زر الحظر (Block)

3. **العودة إلى المتصفح الأول:**
   - يجب أن يحدث التالي تلقائياً:
     - **يتم تشغيل صوت الإشعار 🔊**
     - يظهر SweetAlert مع الرسالة:
       - العنوان: "تم حظر حسابك من قبل الاداره"
       - النص: "تم حظرك من قبل الادارة"
       - زر: "OK"
   - عند الضغط على OK، يتم تسجيل الخروج تلقائياً
   - يتم إعادة التوجيه إلى صفحة تسجيل الدخول

#### النتيجة المتوقعة:
✅ يتم تشغيل صوت الإشعار فوراً 🔊
✅ يتم عرض الإشعار فوراً
✅ يتم تسجيل الخروج تلقائياً
✅ لا يمكن للأدمن المحظور الوصول إلى أي صفحة

---

### الاختبار 2: حذف أدمن متصل (Real-time)

#### الخطوات:
1. **في المتصفح الأول:**
   - سجل دخول بحساب أدمن (مثلاً: admin1@example.com)
   - انتقل إلى أي صفحة في لوحة التحكم
   - اترك الصفحة مفتوحة

2. **في المتصفح الثاني:**
   - سجل دخول بحساب أدمن آخر له صلاحيات
   - انتقل إلى صفحة إدارة المستخدمين/الأدمن
   - ابحث عن admin1
   - اضغط على زر الحذف (Delete)
   - أكد الحذف

3. **العودة إلى المتصفح الأول:**
   - يجب أن يحدث التالي تلقائياً:
     - **يتم تشغيل صوت الإشعار 🔊**
     - يظهر SweetAlert مع الرسالة:
       - العنوان: "تم حذف الحساب"
       - النص: "تم حذف حسابك من قبل الادارة"
       - زر: "OK"
   - عند الضغط على OK، يتم تسجيل الخروج تلقائياً
   - يتم إعادة التوجيه إلى صفحة تسجيل الدخول

#### النتيجة المتوقعة:
✅ يتم تشغيل صوت الإشعار فوراً 🔊
✅ يتم عرض إشعار الحذف فوراً
✅ يتم تسجيل الخروج تلقائياً
✅ لا يمكن للأدمن المحذوف تسجيل الدخول مرة أخرى

---

### الاختبار 3: حظر أدمن غير متصل

#### الخطوات:
1. **في المتصفح الأول:**
   - سجل دخول بحساب أدمن (مثلاً: admin1@example.com)
   - أغلق المتصفح أو علامة التبويب (لكن لا تسجل الخروج)

2. **في المتصفح الثاني:**
   - سجل دخول بحساب أدمن آخر
   - قم بحظر admin1

3. **افتح المتصفح الأول مرة أخرى:**
   - حاول الوصول إلى أي صفحة في لوحة التحكم
   - أو حاول تسجيل الدخول مرة أخرى

#### النتيجة المتوقعة:
✅ إذا كانت الجلسة لا تزال نشطة:
  - يتم اكتشاف الحظر عبر Middleware
  - يتم تسجيل الخروج تلقائياً
  - يتم إعادة التوجيه إلى صفحة تسجيل الدخول مع رسالة خطأ

✅ إذا حاول تسجيل الدخول:
  - يظهر رسالة: "تم حظر حسابك من قبل الاداره"
  - لا يتم السماح بتسجيل الدخول

---

### الاختبار 4: إلغاء الحظر

#### الخطوات:
1. قم بإلغاء حظر الأدمن من لوحة التحكم
2. حاول تسجيل الدخول بحساب الأدمن

#### النتيجة المتوقعة:
✅ يتم السماح بتسجيل الدخول بنجاح
✅ يمكن الوصول إلى جميع الصفحات

## نقاط التحقق

### في الكونسول (Console):
افتح Developer Tools → Console وتحقق من:
```javascript
// عند استقبال الإشعار، يجب أن ترى:
[firebase-messaging-sw.js] Received background message
```

### في قاعدة البيانات:
```sql
-- تحقق من حالة الحظر
SELECT id, name, email, is_blocked FROM admins WHERE email = 'admin1@example.com';

-- تحقق من الإشعارات المرسلة
SELECT * FROM notifications WHERE notifiable_type = 'App\\Models\\AllUsers\\Admin' 
  AND data LIKE '%admin_user_blocked%' 
  ORDER BY created_at DESC LIMIT 5;

-- تحقق من الأجهزة المسجلة
SELECT * FROM devices WHERE morph_type = 'App\\Models\\AllUsers\\Admin' 
  AND morph_id = (SELECT id FROM admins WHERE email = 'admin1@example.com');
```

### في Laravel Log:
```bash
# تحقق من سجلات FCM
tail -f storage/logs/laravel.log | grep -i fcm
```

## استكشاف الأخطاء

### المشكلة: لا يظهر الإشعار أو الصوت
**الحلول:**
1. تحقق من أن Firebase مضبوط بشكل صحيح
2. تحقق من أن `is_notify = 1` للأدمن
3. تحقق من أن الجهاز مسجل في جدول `devices`
4. تحقق من أن ملف الصوت موجود في: `storage/images/settings/notification_sound`
5. تحقق من Console للأخطاء
6. تأكد من أن المتصفح يسمح بتشغيل الصوت التلقائي

### المشكلة: لا يتم تسجيل الخروج
**الحلول:**
1. تحقق من أن route `admin.logout` موجود
2. تحقق من Console للأخطاء في JavaScript
3. تحقق من أن SweetAlert يعمل بشكل صحيح

### المشكلة: يمكن للأدمن المحظور الوصول إلى الصفحات
**الحلول:**
1. تحقق من أن `IsBlockedMiddleware` مسجل في `bootstrap/app.php`
2. تحقق من أن الـ routes محمية بـ middleware `is_blocked`
3. امسح الـ cache:
```bash
php artisan cache:clear
php artisan config:clear
php artisan route:clear
```

## الملفات المهمة للمراجعة

1. **Frontend (JavaScript):**
   - `resources/views/components/admin/firebase.blade.php` - معالج الإشعارات
   - `public/firebase-messaging-sw.js` - Service Worker

2. **Backend (PHP):**
   - `app/Traits/StatusTrait.php` - إرسال الإشعار عند الحظر
   - `app/Http/Middleware/Admin/IsBlockedMiddleware.php` - التحقق من الحظر
   - `app/Jobs/Firebase/SendFirebaseNotificationToMultipleJob.php` - إرسال FCM

3. **Routes:**
   - `routes/web.php` - تحقق من middleware `is_blocked`

4. **Translations:**
   - `lang/ar/notification.php`
   - `lang/en/notification.php`
   - `lang/ar/auth.php`

## ملاحظات إضافية

- الإشعار يُرسل فقط إذا كان `is_notify = 1` للأدمن
- يتم استخدام لغة الجهاز المسجل للأدمن
- إذا كان الأدمن لديه أكثر من جهاز، يتم إرسال الإشعار لجميع الأجهزة
- الـ Middleware يعمل على جميع الطلبات، حتى لو لم يصل الإشعار
- **صوت الإشعار 🔊**: يتم استخدام نفس الصوت المضبوط في إعدادات النظام
- بعض المتصفحات تمنع تشغيل الصوت التلقائي، لكن في هذه الحالة يعمل لأن المستخدم تفاعل مع الصفحة
